### 1. ¿Cómo funciona un modelo de lenguaje (LLM)?
**Concepto:** Un sistema probabilístico que predice la siguiente unidad de información basándose en patrones masivos de datos previos.

### 2. ¿Qué es una API?
**Concepto:** Un puente de software que permite que dos aplicaciones distintas se comuniquen y compartan funciones.
**Analogía:** Es como un **"Menú de restaurante en un centro comercial"**. Tú no necesitas entrar a la cocina del restaurante para obtener la comida; simplemente miras el menú (la documentación), pides un plato específico (la petición) y el personal te lo entrega en tu mesa (la respuesta).

### 3. ¿Qué es el Fine-tuning de un modelo?
**Concepto:** El proceso de entrenar un modelo ya existente con un conjunto de datos específico para que se especialice en una tarea o estilo particular.
**Analogía:** Es como **"Contratar a un abogado brillante y enviarlo a un curso intensivo de especialización en leyes marítimas"**. El abogado ya sabe hablar, escribir y razonar (el modelo base), pero el curso le da el vocabulario técnico y los casos específicos para que sea un experto en un nicho muy concreto.

**Concepto:** Una restricción impuesta por un proveedor para limitar la cantidad de peticiones que un usuario puede hacer en un periodo de tiempo.
**Analogía:** Es como el **"Límite de turnos en una clínica médica"**. Por muy urgente que sea tu consulta, solo puedes entrar si hay un espacio disponible en la agenda del día; si intentas entrar 100 veces por segundo, la puerta simplemente no se abrirá para evitar que el sistema colapse.

### 5. ¿Qué es el Prompt Engineering?
**Concepto:** El arte de refinar las instrucciones dadas a un modelo para obtener el resultado más preciso y útil posible.
**Analogía:** Es como **"Darle instrucciones a un actor de método para que interprete un papel"**.
