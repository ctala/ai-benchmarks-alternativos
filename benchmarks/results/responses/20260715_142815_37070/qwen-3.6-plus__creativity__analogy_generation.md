**1. Cómo funciona un modelo de lenguaje (LLM)**
Un modelo de lenguaje funciona como un maestro sastre que ha analizado millones de patrones de corte y confección: no memoriza prendas, sino que internaliza la lógica estructural de cómo se combinan las piezas para ensamblar, en tiempo real, un texto nuevo que se ajuste exactamente a las medidas que le indicas.

**2. Qué es una API**
Una API actúa como el sistema de contenedores marítimos estandarizados: permite que dos sistemas intercambien datos o servicios sin revelar sus procesos internos, ya que solo deben cumplir el mismo formato de enganche para que la información se cargue, transporte y descargue sin fricciones ni adaptaciones manuales.

**3. Qué es el fine-tuning de un modelo**

**4. Qué es un rate limit**
Un rate limit funciona como el regulador de presión de una red de riego inteligente: si se abren demasiadas válvulas simultáneamente, el sistema pausa temporalmente el flujo para evitar colapsos técnicos y garantiza que cada usuario reciba su cuota justa antes de reabrir el paso de forma controlada.

**5. Qué es el prompt engineering**
El prompt engineering es como la dirección de un teatro de improvisación: no escribes el guion, sino que defines el contexto, las reglas del juego, el tono y el objetivo final, creando un marco preciso que guía a la IA para que genere respuestas útiles, coherentes y alineadas con tu modelo de negocio.
