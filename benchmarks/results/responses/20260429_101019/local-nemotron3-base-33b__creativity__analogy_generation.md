## Respuesta completa

**1. Como funciona un modelo de lenguaje (LLM)**  
Imagina un teatro improvisacional donde el elenco ha estudiado cada obra escrita en la historia y, sin ensayar, puede crear una nueva escena al instante según la solicitud del público. Ese es un modelo de lenguaje: una masa de conocimientos que, al recibir una indicación, conjura una respuesta original sin haberla vivido antes.  

**2. Que es una API**  
Piensa en una API como el camarero de un restaurante que no cocina, pero sabe a qué cocina enviar cada pedido y trae la respuesta al cliente sin que este tenga que entrar en la cocina. Tú envías una solicitud concreta y la API la traduce en la acción adecuada dentro del sistema, devolviéndote el resultado.  

**3. Que es el fine‑tuning de un modelo**  
Es como comprar una camisa lista para usar y, en la medida en que la necesitas, ajustar los puños, el cuello y los botones para que quede perfectamente a tu medida. El fine‑tuning toma un modelo general y lo personaliza con datos específicos, de modo que responda como si estuviera hecho a tu medida.  

**4. Que es un rate limit**  
Imagina un club nocturno con un portero que permite entrar solo a un número limitado de personas por hora, evitando que la fila se vuelva infinita. Un rate limit funciona de la misma forma: controla la cantidad de peticiones que pueden pasar en un período dado, manteniendo el sistema estable.  

**5. Que es el prompt engineering**  
Es como elaborar un mapa del tesoro muy detallado para un explorador; cada pista (palabra) guía al buscador hacia el premio exacto que deseas. Con un prompt bien diseñado, diriges al modelo a producir la respuesta que buscas, minimizando errores y sorpresas.
