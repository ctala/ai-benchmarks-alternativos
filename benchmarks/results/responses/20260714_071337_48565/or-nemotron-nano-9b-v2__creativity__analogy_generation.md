## Respuesta completa



**1. Como funciona un modelo de lenguaje (LLM)**  
Imagina que un LLM es un **chef que cocina una receta sin conocer los ingredientes**. En lugar de seguir pasos fijos, el chef combina sabores, texturas y técnicas de forma creativa, adaptándose a lo que pide el cliente. El modelo "aprende" a partir de miles de recetas (datos) para improvisar el mejor plato (respuesta) posible, sin necesidad de saber cada ingrediente en detalle.  

**2. Que es una API**  
Una API es como un **traductor que solo habla en un idioma específico**. Si quieres pedir un café, no necesitas saber cómo se hace el café, solo dices "un café" y el traductor (API) lo convierte en acciones precise: "extraer café, añadir agua, servir". Es un puente que permite comunicar sin que ambos lados entiendan el mismo lenguaje.  

**3. Que es el fine-tuning de un modelo**  
El fine-tuning es como **entrenar a un perro para que haga solo un truco específico**. En lugar de enseñarle todos los comandos, le das miles de ejemplos de ese truco (ej: "siéntate") hasta que lo domina. El modelo se ajusta para ser más eficiente en esa tarea, como un perro que ya no confunde "siéntate" con "luego".  

**4. Que es un rate limit**  
Un rate limit es como un **barrera de seguridad en un concierto**. Si hay demasiado público, el organizador limita cuántas personas pueden pasar por la barrera cada minuto. Así, evita que se abrumen los recursos (como servidores) y asegura que todos puedan disfrutar sin colapsar el sistema.  

**5. Que es el prompt engineering**  
El prompt engineering es como **escribir una carta de amor con solo 3 palabras**. No se trata de decir "te amo", sino de elegir las palabras exactas que transmitan el mensaje correcto, incluso si el destinatario es un robot. Es el arte de formular preguntas o instrucciones de manera precisa para obtener la mejor respuesta posible.
