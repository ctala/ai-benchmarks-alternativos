2. **Qué es una API**  
Una API es un puente estandarizado que permite que un software le pida algo a otro software y reciba una respuesta sin necesidad de conocer cómo está construido por dentro. Sirve para conectar sistemas de forma rápida, ordenada y automática.  
**Analogía:** es como la ventanilla de una cocina secreta en un restaurante clandestino: tú no entras a los fogones ni tocas las ollas, solo haces tu pedido con un formato claro y te entregan el plato terminado. 3. **Qué es el fine-tuning de un modelo**  
El fine-tuning consiste en volver a entrenar un modelo ya existente con ejemplos más específicos para que responda mejor en una tarea, tono o sector concreto. No se crea desde cero: se afina para que se especialice.  
**Analogía:** es como tomar a un actor brillante de improvisación y encerrarlo dos semanas con vendedores de lujo, médicos o abogados. Sigue siendo el mismo actor, pero ahora domina los gestos, vocabulario y reflejos de ese mundo, y deja de sonar genérico.

4. **Qué es un rate limit**  
Un rate limit es una restricción que limita cuántas veces puedes usar un servicio en un periodo de tiempo. Se pone para evitar abusos, caídas del sistema y para repartir recursos entre todos los usuarios.  
**Analogía:** imagina una discoteca con una máquina de humo espectacular, pero el dueño solo permite activarla ciertas veces por hora. No es porque no quiera fiesta, sino porque si todos aprietan el botón sin control, en cinco minutos nadie ve nada y el show se arruina para todos.

5. **Qué es el prompt engineering**  
El prompt engineering es el arte de formular instrucciones de manera clara y estratégica para obtener mejores respuestas de un modelo. No cambia el modelo en sí: cambia cómo le pides las cosas para guiar mejor el resultado.
