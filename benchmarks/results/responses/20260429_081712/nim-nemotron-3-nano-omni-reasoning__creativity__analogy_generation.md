## Respuesta completa

**1. Como funciona un modelo de lenguaje (LLM)**  
Un modelo de lenguaje es como un DJ que ha escuchado cada canción del mundo; cuando recibe una petición, mezcla fragmentos de esas melodías para crear una pista única al instante. No necesita partitura escrita, solo la intención del usuario para improvisar.

**2. Que es una API**  
Una API es como el botón de pedido de una cafetería automática: tú indicas qué deseas, pulsas el botón y la máquina prepara y entrega exactamente ese producto, sin que tengas que entrar a la cocina. El proceso es rápido, estándar y fiable, sin que tengas que conocer los entresijos del equipo.

**3. Que es el fine‑tuning de un modelo**  
El fine‑tuning es como tomar un pastel genérico y, con ingredientes y decoración específicos, transformarlo en una tarta personalizada para una ocasión especial. El proceso implica entrenar brevemente el modelo con datos nuevos para que adopte el sabor exacto que necesitas.

**4. Que es un rate limit**  
Un rate limit es como el ascensor de un edificio que solo permite subir a tres personas cada minuto; si intentas más, el sistema te hace esperar hasta que llegue el siguiente ciclo. Así se asegura de que todos tengan acceso justo sin saturar el recurso.

**5. Que es el prompt engineering**  
Prompt engineering es como ser el director de una orquesta invisible: con la batuta de tus palabras indicas a los músicos (el modelo) qué notas tocar, logrando la melodía que buscas. Cada detalle de la instrucción actúa como una pista para que el modelo interprete exactamente lo que deseas.
