## Respuesta completa

1. **Cómo funciona un modelo de lenguaje (LLM)**  
Un LLM predice qué palabra o idea tiene más sentido después de otra, usando patrones aprendidos de enormes cantidades de texto. Es como un chef que nunca probó tu plato, pero leyó millones de menús y recetas: no “sabe” con certeza qué quieres, pero puede preparar algo muy probable según los ingredientes que le das.

2. **Qué es una API**  
Una API es una forma controlada de pedirle a un software que haga algo o entregue información, sin tener que entrar a sus entrañas. Es como una ventanilla secreta en un restaurante: no entras a la cocina, pero puedes pedir “quiero el total de ventas de hoy” y la cocina te devuelve el plato exacto.

3. **Qué es el fine-tuning de un modelo**  
El fine-tuning es entrenar un modelo ya existente con ejemplos específicos para que responda mejor en un contexto concreto: tu marca, tu industria o tu estilo. Es como tomar a un actor muy talentoso y hacerlo ensayar durante semanas solo el papel de tu empresa, hasta que habla con tu tono y entiende tus escenas típicas.

4. **Qué es un rate limit**  
Un rate limit es un límite de cuántas veces puedes usar un servicio en cierto periodo de tiempo. Es como una máquina de café de oficina que solo permite preparar 30 cafés por minuto: no importa cuánta gente quiera café, el sistema pone una pausa para no quemarse.

5. **Qué es el prompt engineering**  
El prompt engineering es el arte de pedirle instrucciones al modelo de forma clara, precisa y útil para obtener mejores respuestas. Es como darle una misión a un mago con contrato legal: si dices “haz algo increíble”, improvisa; si dices “crea tres ideas de anuncios para madres primerizas, tono emocional y máximo 20 palabras”, aparece justo lo que necesitas.
