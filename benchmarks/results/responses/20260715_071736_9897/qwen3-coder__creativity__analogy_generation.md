**Concepto**: Una API es una manera de pedirle algo a un sistema (como datos o una acción) sin tener que saber cómo lo hace internamente.

### 3. **¿Qué es el fine-tuning de un modelo?**

**Concepto**: Es entrenar un modelo ya existente para que se vuelva experto en un tema específico.

**Analogía**: Es como contratar a un prodigio del ajedrez y luego entrenarlo intensamente solo en aperturas de gambito de dama. ### 4. **¿Qué es un rate limit?**

**Concepto**: Es un límite en la cantidad de solicitudes que puedes hacerle a un sistema en un tiempo determinado.

**Analogía**: Es como estar en una fiesta VIP donde solo te dejan pasar 5 personas por minuto. Si intentas meter a 20 amigos de golpe, el portero te dice: “Espera un poco, esto no es una estampida”. Es una forma de evitar que el sistema se colapse, aunque tú quieras más velocidad.

### 5. **¿Qué es el prompt engineering?**

**Concepto**: Es el arte de formular instrucciones muy específicas para que un modelo haga exactamente lo que tú quieres.

**Analogía**: Es como darle órdenes a un genio que cumple lo que le pidas al pie de la letra... pero es muy literal. Si le dices “Hazme una casa”, podrías terminar con una casita de juguete. Pero si dices “Constrúyeme una casa moderna de 3 dormitorios en estilo escandinavo con jardín al frente”, el genio entiende que no quieres un castillo ni una choza. El prompt engineer es el que aprende a hablar con precisión al genio.
